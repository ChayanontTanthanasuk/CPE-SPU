# 07 — Internationalization (i18next + react-i18next)

โปรเจคนี้รองรับ **ภาษาไทย** และ **ภาษาอังกฤษ** ผ่าน i18next  
**กฎ:** ข้อความ UI ทุกตัวต้องผ่าน i18n — ห้าม hardcode string ใน JSX

---

## ส่วนที่ 1 — แนวคิด i18n

### 1.1 i18n คืออะไร

i18n (Internationalization) คือการออกแบบแอปให้รองรับหลายภาษาและวัฒนธรรมได้

**ปัญหาถ้าไม่ใช้ i18n:**
```tsx
// ผิด — hardcode ภาษาไทยตรงๆ
<button>บันทึก</button>
<p>ยินดีต้อนรับ, {user.name}</p>
```

**ถูก — ใช้ translation key:**
```tsx
// translation key → ดึงข้อความจาก JSON ตาม language ที่เลือก
<button>{t('common.save')}</button>
<p>{t('greeting', { name: user.name })}</p>
```

---

### 1.2 โครงสร้างไฟล์

```
src/i18n/
├── th.json    ← ภาษาไทย
└── en.json    ← ภาษาอังกฤษ
```

## ส่วนที่ 3 — การใช้งาน

### 3.1 useTranslation Hook

```tsx
import { useTranslation } from 'react-i18next';

function NewsPage() {
  const { t, i18n } = useTranslation();

  return (
    <div>
      <h1>{t('news.title')}</h1>
      <p>{t('news.empty')}</p>

      {/* ภาษาปัจจุบัน */}
      <p>ภาษาปัจจุบัน: {i18n.language}</p>
    </div>
  );
}
```

---

### 3.2 Translation กับ Variables (Interpolation)

```json
// th.json
{
  "greeting": "สวัสดี {{name}}",
  "news": {
    "deleteConfirm": "ต้องการลบ \"{{title}}\" หรือไม่?",
    "count": "{{count}} รายการ"
  }
}
```

```tsx
// ใช้งาน
t('greeting', { name: 'Alice' })
// → "สวัสดี Alice"

t('news.deleteConfirm', { title: news.title })
// → "ต้องการลบ "ข่าวทดสอบ" หรือไม่?"

t('news.count', { count: 42 })
// → "42 รายการ"
```

---

### 3.3 Plural Forms

```json
// en.json
{
  "item": "{{count}} item",
  "item_one": "{{count}} item",
  "item_other": "{{count}} items"
}
```

```tsx
t('item', { count: 1 })  // "1 item"
t('item', { count: 5 })  // "5 items"
```

---

### 3.4 เปลี่ยนภาษา

```tsx
import { useTranslation } from 'react-i18next';

function LanguageSwitcher() {
  const { i18n } = useTranslation();

  const toggleLanguage = () => {
    const nextLang = i18n.language === 'th' ? 'en' : 'th';
    i18n.changeLanguage(nextLang);
  };

  return (
    <button onClick={toggleLanguage}>
      {i18n.language === 'th' ? 'EN' : 'ไทย'}
    </button>
  );
}
```

---

### 3.5 Namespace (สำหรับโปรเจคขนาดใหญ่)

```tsx
// แบ่ง translation ตาม feature
const { t } = useTranslation('news');  // ใช้ namespace 'news'
t('title');    // ไม่ต้องเขียน 'news.title'
t('empty');    // ไม่ต้องเขียน 'news.empty'

// Default namespace
const { t } = useTranslation();         // ใช้ namespace 'common'
t('save');
```

## ส่วนที่ 4 — Workflow การเพิ่ม Text ใหม่

ทุกครั้งที่ต้องการเพิ่มข้อความใหม่:

1. **เพิ่ม key ใน `th.json`** ก่อน
2. **เพิ่ม key เดียวกัน ใน `en.json`**
3. **ใช้ `t('key')` ใน component**

```bash
# ตัวอย่าง — เพิ่มหน้า "คณาจารย์"
```

```json
// th.json
{
  "faculty": {
    "title": "คณาจารย์",
    "name": "ชื่อ-สกุล",
    "position": "ตำแหน่ง",
    "email": "อีเมล",
    "empty": "ไม่พบข้อมูลอาจารย์"
  }
}
```

```json
// en.json
{
  "faculty": {
    "title": "Faculty",
    "name": "Name",
    "position": "Position",
    "email": "Email",
    "empty": "No faculty found"
  }
}
```

```tsx
// faculty/pages/FacultyPage.tsx
const { t } = useTranslation();

return (
  <div>
    <h1>{t('faculty.title')}</h1>
    {faculties.length === 0 && <p>{t('faculty.empty')}</p>}
  </div>

```
